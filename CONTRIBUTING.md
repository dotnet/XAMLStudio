# Contributing

This project welcomes contributions and suggestions. Most contributions require you to
agree to a Contributor License Agreement (CLA) declaring that you have the right to,
and actually do, grant us the rights to use your contribution. For details, visit
https://cla.dotnetfoundation.org.

When you submit a pull request, a CLA-bot will automatically determine whether you need
to provide a CLA and decorate the PR appropriately (e.g., label, comment). Simply follow the
instructions provided by the bot. You will only need to do this once across all repositories using our CLA.

This project has adopted the [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/).
For more information see the [Code of Conduct FAQ](https://opensource.microsoft.com/codeofconduct/faq/)
or contact [opencode@microsoft.com](mailto:opencode@microsoft.com) with any additional questions or comments.

## Questions ❓

If you have general questions about using XAML Studio, please [post in the Q&A discussions area](https://github.com/dotnet/XAMLStudio/discussions/categories/q-a).

## New Features 💡

If you have a new idea (or want to vote on existing ideas), please visit the [ideas discussion area](https://github.com/dotnet/XAMLStudio/discussions/categories/ideas).

There's also the [feature feedback discussion area](https://github.com/dotnet/XAMLStudio/discussions/categories/feature-feedback) looking for any feedback you have on upcoming v2 features.

## Getting Started 🏃‍➡️

Otherwise, to contribute code, please engage with an existing issue or file a new issue before submitting a Pull Request (PR). 

Not sure where to start? 🤔 Easy starting places to look are for issues labeled as [good first issue](https://github.com/dotnet/XAMLStudio/labels/good%20first%20issue) or [help wanted](https://github.com/dotnet/XAMLStudio/labels/help%20wanted).
